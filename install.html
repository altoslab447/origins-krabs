小龍蝦安裝教學

從一台全新 VPS 到完整運行的 AI 助理，手把手完整教學

⚠️ 開始前請確認
  • 一台 VPS（Ubuntu 22.04 LTS 推薦，至少 1GB RAM）
  • 能 SSH 連線到你的伺服器
  • 已取得 AI API Key（如 Gemini、DeepSeek 等）
  • 已取得 Telegram Bot Token（向 @BotFather 申請）
1
Step 01

初始伺服器設定

SSH 連線到你的 VPS 並更新套件列表，確保系統環境就緒。

bash 
# 連線到你的 VPS（替換為你的 IP）
ssh root@YOUR_SERVER_IP
bash 
# 更新套件列表（只更新列表，不全面升級以節省時間）
apt-get update -y
bash 
# 安裝必要基礎工具
apt-get install -y curl wget git build-essential
2
Step 02

安裝 Node.js 20

龍蝦（OpenClaw）基於 Node.js 運行，需要 Node.js 18 或以上版本。推薦使用官方 NodeSource 源安裝 Node.js 20 LTS。

bash 
# 安裝 NodeSource 源
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
bash 
# 安裝 Node.js
apt-get install -y nodejs
bash 
# 確認安裝成功
node --version   # 應顯示 v20.x.x
npm --version    # 應顯示 10.x.x
✅
看到版本號出現就代表安裝成功。Node.js 20 LTS 是目前最穩定的選擇，支援到 2026 年 4 月。
3
Step 03

安裝龍蝦核心套件

OpenClaw 是龍蝦的核心引擎；ClaWHub 是技能管理工具，用來安裝和管理各種功能技能。

bash 
# 全域安裝（需要 root 或 sudo）
npm install -g openclaw clawhub
bash 
# 確認安裝成功
openclaw --version
clawhub --version
⚠️
出現 EACCES 權限錯誤？ 執行以下指令修正 npm prefix 後重新安裝：
bash 
npm config set prefix '/usr/local'
npm install -g openclaw clawhub
4
Step 04

初始化並設定 API Keys

建立設定目錄並完成龍蝦初始化設定，填入你的 AI API Key 和 Telegram Bot Token。

bash 
# 建立設定目錄
mkdir -p /root/.openclaw
bash 
# 初始化設定（互動式）
openclaw init
💡
執行後會互動式詢問你：
· AI Provider（選擇 gemini / openai / anthropic / deepseek 等）
· AI API Key
· Telegram Bot Token
· 其他進階設定（可直接按 Enter 跳過）
或者手動建立設定檔
bash 
# 手動建立設定檔
cat > /root/.openclaw/config.json << 'EOF'
{
  "ai": {
    "provider": "gemini",
    "api_key": "YOUR_GEMINI_API_KEY",
    "model": "gemini-2.0-flash"
  },
  "telegram": {
    "bot_token": "YOUR_BOT_TOKEN"
  },
  "streaming": false,
  "tools": {
    "deny": []
  }
}
EOF
⚠️
記得將 YOUR_GEMINI_API_KEY 和 YOUR_BOT_TOKEN 替換為你實際的值。設定完成後這個檔案請妥善保管，不要公開分享。
5
Step 05

安裝推薦技能

大閘蟹建議安裝以下 9 種技能，覆蓋日常所需的搜尋、天氣、股票、加密貨幣等功能。

bash 
# 安裝不需要 --force 的技能
clawhub install ddg-web-search weather summarize translate youtube-transcript
bash 
# 安裝需要 --force 的技能（VirusTotal 誤判，安全可用）
clawhub install --force skill-creator stock-info-explorer news-summary crypto-market-data
📦
為什麼需要 --force？ 部分技能的腳本可能被 VirusTotal 等安全掃描工具誤判。這些技能完全安全，加上 --force 旗標僅是告訴 ClaWHub 跳過安全警告繼續安裝。
🔧
stock-info-explorer 需要額外安裝 uv（Python 環境管理工具）：
bash 
# 安裝 uv（stock-info-explorer 的 Python 依賴）
curl -LsSf https://astral.sh/uv/install.sh | env HOME=/root sh
ln -sf /root/.local/bin/uv /usr/local/bin/uv
6
Step 06

設定 systemd 服務（開機自動啟動）

透過 systemd 服務讓龍蝦在伺服器重啟後自動啟動，並在崩潰時自動重啟。

bash 
# 建立 systemd service 檔案
cat > /etc/systemd/system/openclaw.service << 'EOF'
[Unit]
Description=OpenClaw AI Agent
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root
ExecStart=/usr/local/bin/openclaw start
Restart=always
RestartSec=10
Environment=HOME=/root

[Install]
WantedBy=multi-user.target
EOF
bash 
# 啟用並啟動服務
systemctl daemon-reload
systemctl enable openclaw
systemctl start openclaw
bash 
# 確認服務狀態
systemctl status openclaw
✅ 看到 Active: active (running) 表示成功！
7
Step 07

測試你的大閘蟹

一切設定完成後，打開 Telegram 測試你的 AI 助理是否正常運行。

  • 1 打開 Telegram，搜尋你的 Bot username
  • 2 傳送 /start 開始對話
  • 3 試試說「你好」或「今天天氣如何？」
  • 4 如果有回應，恭喜！安裝成功 🎉

常見問題排查

  • Bot 沒有回應 → 確認 Bot Token 正確，確認服務正在運行（systemctl status openclaw）
  • API 錯誤 → 確認 API Key 正確且有剩餘額度
  • 服務崩潰 → 查看日誌（journalctl -u openclaw -f）
  • 權限問題 → 確認以 root 身份執行，或使用 sudo

查看日誌

bash 
# 查看即時日誌
journalctl -u openclaw -f

# 查看最近 100 行日誌
journalctl -u openclaw -n 100

設定完成！接下來呢？
